Transcript pipeline

Podible now treats chapter analysis as timestamped audio transcription. It no longer projects Whisper output onto canonical EPUB words or derives chapter markers from EPUB structure.

Input

Preferred audio asset, optional EPUB for glossary hints, book metadata, and OpenAI settings.

Output

A persisted transcript JSON file containing raw Whisper text and word-level timestamps.

Chapters

Served from embedded M4B chapters or multi-file audio boundaries, not transcript alignment.

Why the alignment layer was removed

The old pipeline attempted to align noisy transcript tokens to EPUB tokens, then generate synthetic chapter boundaries and canonical timestamped EPUB words. That produced a lot of code, high CPU and memory pressure, and brittle chapter markers. The practical value was low compared with directly serving Whisper's word-level timestamp output.

gpt-4o-transcribe currently does not expose word or segment timestamps through the transcription API. Because timestamps are a hard requirement, Podible uses whisper-1 for this path.

End-to-end flow

  1. The importer or scanner queues a chapter_analysis job for the preferred audio asset.
  2. The job fingerprints the audio files and reuses a matching persisted transcript when one exists.
  3. If an EPUB is available, Podible extracts a compact glossary of names and terms for the Whisper prompt.
  4. The audio is split into overlapping chunks and each chunk is extracted with ffmpeg.
  5. Each chunk is sent to whisper-1 with verbose_json and word timestamps enabled.
  6. Chunk results are persisted immediately to temporary JSON files, then merged after all chunks finish.
  7. The merged transcript is written under the config transcript directory and referenced by SQLite.
  8. /transcripts/:assetId.json file-serves the persisted artifact.

Transcript shape

{
  "version": "1.4.0",
  "text": "full raw transcript text",
  "words": [
    { "startMs": 0, "endMs": 420, "text": "Hello", "token": "hello" }
  ],
  "chunks": [
    {
      "index": 0,
      "startMs": 0,
      "durationMs": 1800000,
      "trimStartMs": 0,
      "trimEndMs": 15000,
      "wordStartIndex": 0,
      "wordEndIndex": 5120,
      "text": "chunk text"
    }
  ]
}

The public payload omits legacy canonical EPUB segments. Old aligned transcript artifacts can still exist on disk until the audio fingerprint or algorithm version causes a new transcript run.

Runtime boundaries

src/library/chapter-analysis.ts owns EPUB glossary loading, chunk extraction, Whisper calls, chunk merging, transcript artifact persistence, and job bookkeeping. The deleted alignment worker files are no longer part of production.

src/library/media.ts owns chapter JSON. It reads embedded chapters for single-file audio and file boundaries for multi-file audio. Transcript jobs do not mutate chapter output.
