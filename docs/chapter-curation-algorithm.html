Chapter Curation Algorithm

Recursive agentic boundary search for audiobook chapters

Goal

Given an audiobook that has both an EPUB and one or more audio assets, produce a list of chapter start timestamps — one per EPUB spine node that is actually read aloud — expressed in seconds from the beginning of the combined audio.

The core challenge is that audiobook narrators do not always read chapter headings, ASR sometimes drops words, and multi-file audiobooks require combining transcripts with proper time offsets. A simple text-search approach fails often enough that the task is delegated to LLM agents that can reason about evidence quality and retry with different strategies.

Input
EPUB + audio
Spine nodes & Whisper transcript utterances
Output
chapters_json
[{title, startTime}] stored on the first audio asset's chapter_analysis row

Pipeline integration

Curation is triggered automatically as part of the chapter_analysis background job, which runs once per audio asset after its Whisper transcription completes.

Algorithm overview

The algorithm runs in three sequential phases:

Phase 1 — EPUB node classification

The AudibleEpubNodeClassifier agent reviews all EPUB spine nodes and the transcript start/end excerpts and filters out nodes that are not spoken in the audio (copyright pages, dedications, table of contents, about-the-author, etc.). It also annotates any audio-only intervals — spans of audio (publisher intros, end credits, part bumpers) that have no corresponding EPUB node.

The filtered node list becomes the working set for Phase 2. False exclusions are penalised more than false inclusions, so the agent errs toward keeping uncertain nodes.

Phase 2 — Recursive binary splitting

The algorithm builds a binary span tree over the filtered EPUB nodes and the audio timeline. Each span covers a contiguous range of EPUB nodes and a corresponding time range. The root span covers all nodes from t=0 to the total audio duration.

At each non-leaf span the algorithm:

  1. Picks a target boundary — the EPUB node whose start timestamp is most uncertain, ranked by proximity to the midpoint of the span's node range.
  2. Attempts a deterministic fast path (see below) before calling the agent.
  3. If deterministic fails, runs the SectionChapterCurator agent to find the timestamp.
  4. The curator's proposed timestamp is validated by the ChapterBoundaryJudge agent.
  5. If accepted, the span is split at that timestamp: the left child covers nodes before the boundary, the right child covers nodes from the boundary onward.
  6. Left and right children are resolved concurrently (up to 24 spans in-flight at once).
  7. If the agent fails or the boundary is rejected, the span becomes a partial leaf and its nodes are divided proportionally by EPUB word count.
The target boundary is always the node closest to the midpoint of the span (by node index). This minimises the worst-case tree depth — a binary search pattern — so a book with N chapters needs at most log₂(N) levels of recursion.

Phase 3 — Output assembly

Leaf spans each contribute one chapter entry: the boundary's confirmed timestamp (or the span's inherited start time if no boundary was found). The full list is deduplicated, sorted, and serialised as [{title: string, startTime: number}] in seconds.

Agents

Three agents are used, all backed by the OpenAI Agents SDK with the model configured in Podible settings (defaulting to a GPT-4-class model). Tracing is sent to the OpenAI platform using the same API key.

AudibleEpubNodeClassifier Phase 1

One-shot agent. Reviews EPUB spine nodes and transcript excerpts to decide which nodes are plausible audiobook chapter material. Runs once per book, before the recursive split phase begins.

System prompt goals

  • Exclude obvious non-audio front/back matter (copyright, dedication, ToC, nav, ads, index, about-author).
  • Keep all uncertain narrative nodes — false exclusions are worse than false inclusions.
  • Identify audio-only intervals (publisher intros, credits, part bumpers) not covered by any EPUB node.

Per-call prompt contents

  • Book title and author
  • First ~4,000 chars of transcript
  • Last ~4,000 chars of transcript
  • Boundary excerpts around each asset start, end, and join point (for multi-asset books)
  • All EPUB spine nodes: id, title, href, word count, first 50 words

Tool surface

ToolDescription
submitAudibleEpubNodeSelection Terminal tool. Submits the list of included node IDs and any audio-only intervals.

Max turns: 4

SectionChapterCurator Phase 2

One agent instance per span, instantiated fresh for each boundary search. Its sole job is to find the timestamp where one specific assigned EPUB node begins in the audio. It has access to both the EPUB text and the transcript and must gather evidence before submitting.

System prompt goals (10-step workflow)

  1. Use targetBoundaryContext from the prompt as the starting EPUB opener context.
  2. Call researchEpubBoundary first — it searches the assigned node automatically.
  3. If research returns a strong opener/near_opener, use that timestamp.
  4. Otherwise call getEpubNodeText and try different opener phrases from the same node.
  5. Estimate likely position from EPUB word-count ratio; search nearby with rgSearchTranscript.
  6. Inspect the best candidate with getTranscriptWindow (radiusSeconds=45 after a rejection).
  7. If context looks like pre-target or interior prose, call searchEpubText to classify the phrase.
  8. submitBoundarySplit is a final evidence-backed claim — not a guess.
  9. Only submit once opener evidence is confirmed; include proof in evidence/notes fields.
  10. After any rejection, run a fresh transcript search before retrying.

Per-call prompt contents

  • Book title and author
  • Current span: time range, EPUB node range, path in tree
  • Target boundary: EPUB node id, title, expected timestamp, first/last words
  • Neighbouring node context (previous tail text, next head text)
  • Embedded audio chapter markers (if present in the audio file)
  • Any audio-only intervals identified by the classifier

Tool surface

ToolDescription
researchEpubBoundary All-in-one research tool. Generates opener phrase variants from the assigned EPUB node, searches the transcript in a radius around the expected time, returns hits, windows, and reverse classification (opener / near_opener / interior / pre_target / post_target).
getEpubNodeText Returns full or windowed text of the assigned EPUB node plus exact-search phrase variants. Locked to the target node — cannot retrieve other nodes.
searchEpubText Reverse-search: given a transcript phrase, classifies where it falls relative to the target node (opener, near_opener, interior, pre_target, post_target).
rgSearchTranscript Ripgrep search over transcript utterances within this span. Accepts a time scope (seconds).
fuzzySearchTranscript Fuzzy search over transcript utterances within this span. Useful when ASR has transcription errors.
getTranscriptWindow Returns transcript utterances around a specific timestamp within this span. Used to inspect a candidate before submitting.
submitBoundarySplit Terminal tool. Submits the confirmed start timestamp for the assigned EPUB node. Triggers judge validation.

Max turns: 64  ·  On max-turns error, advances to the next-closest target boundary candidate and retries with a fallback model.

ChapterBoundaryJudge Phase 2

Called once per curator submission. Reviews the proposed timestamp against a pre-built boundary comparison object (transcript before/after, EPUB tail/head text, precision metadata) and accepts or rejects. Does not suggest alternatives — only accepts or rejects.

System prompt goals

  • Compare previousEpub.tailText to transcriptBefore and targetEpub.bodyHeadText to transcriptAfter.
  • Accept when transcriptBefore matches the previous EPUB tail and transcriptAfter begins with the target body opener, even if the printed heading is absent from ASR.
  • Reject when opener evidence is absent, offset, generic, interior, or only a near-match.
  • Handle word-precision vs utterance-precision timestamps differently (utterance-precision means the boundary may fall inside a displayed utterance).
  • Use structured evidence terms in rejection reasons, not narrative terms.

Per-call prompt contents

  • Book title and author
  • The curator's proposed epubNodeId, title, startTime, and notes
  • Boundary comparison: previous EPUB tail text, target EPUB head/body text, optional heading text
  • Transcript window before and after the proposed timestamp
  • Evidence candidates with quality scores
  • Transcript precision (word or utterance) and nearest clean boundary times

Tool surface

ToolDescription
submitBoundaryJudgment Terminal tool. Submits accepted: true/false, an evidence classification, and optional notes.

Max turns: single-turn (no iterative tool use)

Span tree mechanics

Span data structure

Each span carries:

  • epubStartIndex / epubEndIndex — index range into the filtered EPUB node array
  • startTime / endTime — time range in seconds
  • depth — tree depth (root = 0)
  • path — human-readable path: root, L, R, LL, LR, …
  • startBoundary — the confirmed split that created this span's left edge (if any)

Target boundary selection

Candidate boundaries are all EPUB nodes strictly between epubStartIndex and epubEndIndex. They are ranked by how close they are to the midpoint of the node index range, with ties broken by index order. The closest-to-midpoint node is selected first, giving a balanced binary search pattern.

Leaf conditions

  • Auto-leaf: Span has only one EPUB node (no internal boundaries). Contributes one chapter at the span's start time.
  • Partial leaf: Span has internal boundaries but curation failed (no decision, wrong target, or agent error). All internal nodes get timestamps proportional to their EPUB word-count position within the span.

Concurrency

Left and right child spans are resolved concurrently using Promise.all. A semaphore caps active agent calls at 24 to avoid rate-limit errors. With a balanced tree this means many spans at the same depth run in parallel.

Deterministic fast path

Before invoking the curator agent, the algorithm tries to resolve a boundary without an LLM call:

  1. Embedded audio chapters: If the audio file itself contains chapter markers (e.g. from an Audible M4B), and the embedded marker can be matched to the target EPUB node with high confidence, that timestamp is used directly.
  2. Inherited boundary tolerance: If the target boundary's expected timestamp is within 2 seconds of an already-confirmed boundary from a sibling span, the confirmed timestamp is inherited rather than re-searched.
  3. Spoken heading search: A quick ripgrep + fuzzy search for the target node's heading text near the expected position. If a high-confidence match is found and it passes a lightweight plausibility check (not mid-sentence, not interior prose), it is used without calling the agent.
The deterministic path is tried first on every span. For books with reliable embedded chapters or clear spoken headings, curation may complete with zero or very few agent calls.

Storage

Results are stored on the first audio asset's chapter_analysis row in the SQLite database:

  • chapters_json — JSON array of {title: string, startTime: number} objects, times in seconds from the start of the combined audio.
  • status — set to succeeded after curation completes (or failed on error).

The buildTranscriptProposedChapterTimings function in media.ts reads this row and converts it into ChapterTiming[] objects (with startMs/endMs) for use by the media layer and the app's chapter display.

Debug trace files for each span are written to the library root during a curation run, allowing post-hoc inspection of agent inputs, tool calls, and decisions.
