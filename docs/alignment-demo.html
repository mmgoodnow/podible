Fuzzy Dynamic Programming Alignment

This demo shows how a noisy audiobook transcript can be aligned to cleaner EPUB text. The transcript provides timestamps. The book text provides canonical wording and paragraph structure. A banded dynamic-programming pass decides whether each token is a match, fuzzy substitution, insertion, or deletion, then projects timing back onto the book text.

Reading The Output

Match Exact normalized token match.
Fuzzy Small edit-distance or morphological match.
Gap Word exists only on one side.
Projection EPUB tokens inherit transcript timing.

Banded DP Matrix

Rows are EPUB tokens. Columns are transcript tokens. Play the animation to watch the diagonal band sweep across the matrix, score cells, and then backtrack through the chosen path.

What the band means: we do not score the whole table. For each EPUB row, we only examine transcript columns near the expected diagonal so the algorithm stays fast.
What each fill step compares: diagonal = align these two tokens, up = skip the EPUB token, left = skip the transcript token.
How the score works: every cell compares three candidate totals: diag = previous diagonal + match score, up = previous row + gap penalty, and left = previous column + gap penalty. Exact matches add the most, fuzzy matches add less, and gaps or hard mismatches subtract from the running total.
What happens when ASR is wrong: if one word is misheard, a fuzzy diagonal match can still win. If several words are wrong in a row, the path will usually take a few gap steps, then reconnect once the texts start matching again.
What backtracking does: once the best final score is found, the algorithm walks backward through the chosen moves to recover the final alignment path.
